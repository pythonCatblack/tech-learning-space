相机是无人机上最轻便、成本最低、信息量最大的传感器——一张图像可以包含数十万个特征点、丰富的纹理和语义信息。但它的根本局限在于：从二维图像恢复三维结构是一个"病态问题"（ill-posed problem）。理解这个局限，才能理解为什么视觉里程计（VO）需要融合IMU——单目视觉缺少尺度信息，而IMU可以提供这个缺失的维度。

3.1 相机模型：从光子到像素的物理过程

3.1.1 针孔模型（Pinhole Camera Model）

最基础的相机模型是针孔模型——假设所有光线通过一个无穷小的"针孔"后投影到成像平面上。这个模型虽然简化，但包含了理解视觉几何的所有核心要素。

在针孔模型中，三维世界中的一个点 P = [X, Y, Z]ᵀ 通过相机投影到图像平面上的点 p = [u, v]ᵀ，满足以下关系（称为针孔投影方程）：

🔑 相机投影方程（内参模型）

设焦距为 $f$，相机光心在像素坐标系的偏移为 $(c_x, c_y)$，则：

$u = f_x \cdot (X / Z) + c_x$
$v = f_y \cdot (Y / Z) + c_y$

写成矩阵形式（齐次坐标）：$Z \cdot p = K \cdot [R|t] \cdot P$
其中 $K$ 称为内参矩阵（Intrinsic Matrix），包含焦距和光心偏移：

$$K = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$$

⚠️
物理直觉：为什么是 Z 在分母？因为相机镜头将远处物体"压缩"到成像平面——物体距离越远，在图像上的位置越靠近光心。这个除法操作是投影的核心，也是从 2D 恢复 3D 困难的根本原因：不同的 (X, Y, Z) 组合可能投影到同一个 (u, v) 像素位置上。

3.1.2 内参标定：你的相机"焦距"到底是多少？

相机的内参不是固定的——镜头生产公差、安装偏差、温度变化都会导致内参漂移。相机标定（Camera Calibration）就是通过拍摄已知图案（如棋盘格）来精确测量内参的过程。

📐

标定方法

Zhang的棋盘格方法（2000）是最成熟方案，OpenCV和Kalibr工具链都支持。需要拍摄10-20张不同角度的棋盘格图像。

🎯

标定参数

内参 (fₓ,fᵧ,cₓ,cᵧ) + 畸变系数 (k₁,k₂,p₁,p₂,k₃)

3.1.3 镜头畸变：直线为什么会弯曲？

实际镜头不是理想的针孔，存在径向畸变（边缘拉伸）和切向畸变（透镜倾斜）。畸变参数也是标定的一部分。

桶形畸变 (k₁>0)：边缘向外拉伸，广角镜头常见 枕形畸变 (k₁<0)：边缘向内收缩，长焦镜头常见 切向畸变：透镜与sensor不平行导致
🔑 畸变校正公式

畸变校正分为两步：

步骤1 - 归一化坐标： $x = (u - c_x)/f_x$, $y = (v - c_y)/f_y$

步骤2 - 径向畸变校正：
$x' = x(1 + k_1 r^2 + k_2 r^4 + k_3 r^6)$
$y' = y(1 + k_1 r^2 + k_2 r^4 + k_3 r^6)$
其中 $r^2 = x^2 + y^2$

步骤3 - 切向畸变校正：
$x'' = x' + 2p_1 xy + p_2(r^2 + 2x^2)$
$y'' = y' + p_1(r^2 + 2y^2) + 2p_2 xy$

步骤4 - 投影回像素坐标：
$u_{corrected} = f_x \cdot x'' + c_x$
$v_{corrected} = f_y \cdot y'' + c_y$

▼📐 深度：畸变校正完整公式推导

本节给出畸变校正公式的完整推导过程，供想深入理解的读者参考。

步骤1 - 归一化坐标：将像素坐标转换到归一化相机平面（以焦距为单位）：

x = (u - cₓ) / fₓ
y = (v - cᵧ) / fᵧ
r² = x² + y²

步骤2 - 径向畸变校正：实际归一化坐标与理想坐标之间的关系为：

x_distorted = x · (1 + k₁·r² + k₂·r⁴ + k₃·r⁶)
y_distorted = y · (1 + k₁·r² + k₂·r⁴ + k₃·r⁶)

步骤3 - 切向畸变校正：透镜与sensor不平行导致的切向畸变：

x_final = x_distorted + 2·p₁·x·y + p₂·(r² + 2·x²)
y_final = y_distorted + p₁·(r² + 2·y²) + 2·p₂·x·y

步骤4 - 重投影回像素坐标：

u_corrected = fₓ · x_final + cₓ
v_corrected = fᵧ · y_final + cᵧ

💡 工程实现注意：实际应用中通常只保留 k₁（主要项），k₂/k₃ 用于广角镜头。切向畸变 p₁/p₂ 通常较小，必要时再估计。

3.2 相机类型：不是所有相机都一样

3.2.1 单目相机（Monocular）

最简单、成本最低的方案——只有一个摄像头。单目视觉的核心问题是：无法从单帧图像恢复深度。因为投影过程丢失了Z轴（深度）信息。

⚠️
单目视觉的尺度问题：单目VO/SLAM可以恢复相对轨迹（形状），但无法恢复绝对尺度。例如：你可以说"机器人走了10个单位"，但不知道这个"单位"是厘米还是米。融合IMU（积分加速度获取尺度）或引入其他传感器（深度相机/Lidar）是解决尺度问题的常见方法。这也是第11章 VIO 之所以要融合 IMU 的核心原因。

3.2.2 双目相机（Stereo）

双目相机利用两个摄像头之间的视差（Disparity）恢复深度——同一个三维点投影到左右两个图像时，像素位置会有一个水平偏移（视差），这个偏移量与深度成反比。

🔑 三角测量：视差如何变成深度

基线距离（两个相机光心之间的距离）为 $b$，焦距为 $f$。对于三维空间点 $P$，投影到左图像素 $(u_l, v)$，右图像素 $(u_r, v)$（假设已校正，y坐标相同），视差 $d = u_l - u_r$。

深度 $Z = f \cdot b / d$

视差越小（物体越远），深度估计越不准确。通常有效测量范围是基线的10-100倍：基线10cm的双目，有效测距约1-10米。

推导： 由相似三角形：$Z/b = f/d \Rightarrow Z = f \cdot b / d$

3.2.3 深度相机（RGB-D）

RGB-D相机同时输出彩色图和深度图。深度获取方式有两种：

⏱️

ToF（飞行时间）

发射红外光脉冲，测量光从发射到返回的时间。Intel RealSense D455、Microsoft Azure Kinect采用此方案。精度随距离下降，通常有效范围0.1-10m。户外阳光干扰大（红外信噪比下降）。

📊

结构光（Structured Light）

发射已知红外图案，比较返回图案的变形计算深度。Apple Face ID、Intel RealSense D400系列使用。室内效果好，室外阳光干扰严重。

3.2.4 事件相机（Event Camera）

事件相机（Event Camera，如 Prophesee、Celex）是近年来兴起的"类视网膜"传感器。与传统相机输出固定帧率图像不同，事件相机只输出亮度变化的事件——每个像素异步检测亮度变化，输出"变化发生时"的时间和位置。

3.3 计算机视觉基础算法

3.3.1 特征点检测与描述

特征点（Feature）是图像中具有显著性的像素位置——它们在不同帧之间具有可重复性和独特性。传统视觉里程计的核心就是"检测特征点 → 匹配特征点 → 恢复相机运动"。

🔍

FAST 角点

快速检测局部像素值与圆周邻域差异显著的点。速度极快，适合实时应用。缺点：不带描述子，需搭配其他描述子使用。

🎯

Harris 角点

基于图像梯度的角点响应函数，对旋转不变。ORB-SLAM等经典系统使用。

🔢

ORB 描述子

ORB（Oriented FAST and Rotated BRIEF）是FAST+BRIEF的加速版，带方向信息。计算极快，广泛用于实时VIO系统。

🧬

SIFT / SURF / AKAZE

更高精度的描述子。SIFT精度最高但计算慢；AKAZE提供精度与速度的良好平衡。

3.3.2 光流法（Optical Flow）

光流描述图像中像素随时间的运动模式。与特征点法不同，光流利用稠密或半稠密的信息，对低纹理区域更鲁棒。

LK光流：稀疏光流，适合特征点跟踪，计算高效 Horn-Schunck：全局稠密光流，引入平滑约束 Farnebäck：稠密光流，用于运动分割和场景流估计

3.3.3 深度学习视觉感知

2015年后，深度学习在视觉感知领域全面超越传统方法。当前主流网络架构：

🎯

目标检测

YOLO系列（YOLOv5-v8）、Faster R-CNN——实时检测障碍物、人、车。YOLOv8可做到30-100fps。

🔢

语义分割

DeepLabv3+、SegFormer——像素级分类，理解可飞行区域。输出每个像素的类别（天空/地面/障碍物）。

📐

单目深度估计

MiDaS、DPT——从单张图像预测稠密深度图。精度不如Lidar，但无尺度约束，部署灵活。

🔮

场景流/光流

RAFT（Optical Flow）、FlowNet——预测像素级运动矢量，用于运动检测。

边缘部署的挑战

为什么深度学习在无人机上部署困难？

深度学习模型通常需要GPU加速，典型模型（YOLOv8-L）的推理需要10-30 GFLOPS。在Jetson Orin上可以实时运行，但功耗会显著增加。更重要的是：模型训练需要大量标注数据，数据偏差会导致意想不到的失效——在东北雪地训练的模型可能在云南竹林中完全失效。融合传统几何方法（如Lidar几何约束）可以显著提升鲁棒性。这也是第12章融合视觉与Lidar的动机之一。

3.4 双目校正与立体匹配

双目立体匹配的核心任务是：找到左图和右图中对应的像素点，计算视差，进而恢复深度。

3.4.1 极线约束：搜索空间的降维

如果没有约束，在右图中搜索匹配点需要在整个二维图像搜索——这既慢又不准。极线约束（Epipolar Constraint）把这个搜索降到一维：如果两个相机已经完全校正（极线平行），则左图中一个像素对应的点在右图中必然出现在同一行（扫描线）上。

3.4.2 立体匹配算法

⚡

SGBM (Semi-Global Block Matching)

半全局块匹配，速度与精度的良好平衡。OpenCV内置，NVIDIA GPU可硬件加速。实时应用首选。

🎯

AD-Census

精度更高但更慢，适合离线处理（如无人机测绘后重建）。

🤖

深度学习立体匹配

HM3D-Stereo、CASSTEREO等网络，精度显著超越传统方法，但需要GPU实时运行。

🛠️ 动手实践

用OpenCV亲手实现相机标定

目标：用Python+OpenCV对相机进行内参标定，获得 (fₓ, fᵧ, cₓ, cᵧ) 和畸变系数。

步骤：
① 打印棋盘格标定板（OpenCV文档中有标准PDF），拍摄15-20张不同角度的照片。
② 使用以下代码提取角点并标定：

import cv2
import numpy as np

# 准备标定板的世界坐标（棋盘格角点）
objp = np.zeros((9*6, 3), np.float32)
objp[:, :2] = np.mgrid[0:9, 0:6].T.reshape(-1, 2)

objpoints = []  # 3D points
imgpoints = []  # 2D points

for fname in images:  # 你的棋盘格图像列表
    img = cv2.imread(fname)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    ret, corners = cv2.findChessboardCorners(gray, (9,6), None)
    if ret:
        objpoints.append(objp)
        # 亚像素精度优化
        criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
        corners2 = cv2.cornerSubPix(gray, corners, (11,11), (-1,-1), criteria)
        imgpoints.append(corners2)

# 标定！
ret, mtx, dist, rvecs, tvecs = cv2.calibrateCamera(objpoints, imgpoints, gray.shape[::-1], None, None)
print(f"内参矩阵K:\n{mtx}")
print(f"畸变系数: {dist}")
③ 验证标定精度：计算重投影误差，rms < 1.0 像素代表标定良好。
④ 用 cv2.undistort() 校正图像，观察畸变是否消除。

⚠️
常见陷阱① - 棋盘格拍摄角度不足：如果所有照片都是从正前方拍摄（旋转角度不够），标定结果会低估畸变系数。正确做法：让棋盘格出现在图像的四个角落和边缘区域，这些区域畸变最明显。

常见陷阱② - 标定后不及时验证：拿到内参矩阵后，必须用undistort()校正一张训练集外的照片，观察边缘直线是否变直。如果直线仍然弯曲，说明标定数据不足。

常见陷阱③ - 忽视畸变模型版本：OpenCV默认使用5参数畸变模型(k₁, k₂, p₁, p₂, k₃)，但某些广角镜头（>150°）需要OpenCV 4.x+的rational模型(8参数)。否则即使标定误差小，边缘区域校正也不准确。
