传感器融合的核心任务是：从充满噪声和不确定性的传感器数据中，估计出尽可能准确的状态（位置、姿态、速度）。概率论提供了描述和处理不确定性的数学语言——传感器噪声、模型偏差、环境干扰……这些"不确定"的东西，都可以用概率分布来精确描述。本章的目标是让你不仅理解这些概念，还能用Python亲手实现它们。

8.1 概率论核心：高斯分布与协方差矩阵

8.1.1 为什么传感器误差通常用高斯分布？

根据中心极限定理（Central Limit Theorem），大量独立随机因素叠加的结果趋近于正态分布。传感器噪声通常由大量微小、独立的热噪声、量化噪声、振动噪声等叠加而成——因此高斯分布是传感器噪声的合理近似模型。

🔑 一维高斯分布

随机变量 x 服从均值 μ、标准差 σ 的高斯分布：

p(x) = (1/√(2πσ²)) · exp(-(x-μ)²/(2σ²))

其中：
• μ（均值）：最可能的值
• σ²（方差）：不确定性——σ越大，不确定性越高

物理直觉：想象扔飞镖。飞镖落点受风、手抖、目标晃动等几十个独立因素影响——每个因素单独看都很微小，但叠加后落点分布恰好是正态分布。传感器噪声同理。

8.1.2 协方差矩阵：多维不确定性的表示

当状态是向量（如三维位置[x, y, z]ᵀ）时，不确定性需要用协方差矩阵（Covariance Matrix）描述：

🔑 协方差矩阵

协方差矩阵 Σ 是对称正定矩阵：Σ = E[(x-μ)(x-μ)ᵀ]

• 对角线元素：每个维度的方差（Var(x), Var(y), Var(z)）
• 非对角线元素：维度之间的相关程度（Cov(x,y), Cov(y,z), Cov(x,z)）

在二维平面上，协方差矩阵决定高斯分布的误差椭圆——长轴是最大不确定方向。

实战含义：如果x方向和y方向的协方差>0，说明这两个方向的误差倾向于同向偏离（正相关）；如果<0，说明倾向于反向偏离。

8.1.3 协方差传播定律

若 y = Ax + b（线性函数），则协方差传播为：

Σ_y = A · Σ_x · Aᵀ

这是卡尔曼滤波中最重要的公式之一——线性变换后，不确定性也被"变换"放大或缩小。如果A的某行元素很大，相应的方差会被放大。

🎯 协方差传播的实战应用

为什么旋转矩阵不放大误差，而缩放会？

设二维位置 x = [px, py]ᵀ，协方差矩阵 Σx = [σ²x 0; 0 σ²y]。如果绕原点旋转90°（A = [0 -1; 1 0]），则 Σy = AΣxAᵀ = Σx——旋转不改变方差大小，只是换了坐标系方向。

但如果缩放2倍（A = 2I），则 Σy = 4Σx——方差被放大4倍（标准差放大2倍）！这就是为什么相机的内参f如果标定误差10%，三角测量的深度误差就会放大10%的道理。

8.2 贝叶斯滤波：状态估计的统一框架

8.2.1 贝叶斯公式的核心思想

贝叶斯滤波是所有传感器融合算法的理论基石。它的核心思想是：用新的观测数据更新对状态的信念。

🔑 贝叶斯公式

P(x | z) = P(z | x) · P(x) / P(z)

其中：
• P(x)（先验）：观测前对状态 x 的信念
• P(z | x)（似然）：在状态 x 下观察到数据 z 的概率
• P(x | z)（后验）：观测 z 后对状态 x 的更新信念
• P(z) 是归一化常数（确保 P(x|z) 积分为1）

直觉：贝叶斯公式告诉我们：新的观测 z 如何改变我们对状态 x 的认知。如果 P(z|x) 很大（即"如果真的在状态x，观测到z的概率很高"），那么 P(x|z) 也会变大。

8.2.2 预测-更新循环

贝叶斯滤波交替执行两步：

预测步
根据系统模型
预测状态分布
→
更新步
根据新观测
修正状态分布
→
预测步
下一时刻
继续预测...

预测步（Prediction）：利用系统运动模型（如IMU积分），从上一时刻的后验估计预测当前时刻的先验估计。预测会增加不确定性（噪声累积）。

更新步（Update）：利用新的传感器观测，根据贝叶斯公式将先验更新为后验。更新会减少不确定性（观测带来信息）。

8.3 信息论视角：熵与信息增益

理解"置信度"和"信息增益"的概念，对设计鲁棒的传感器融合系统至关重要。

📈

预测增加熵

IMU积分使位置估计的不确定性（熵）随时间增长，误差椭圆越来越大。

📉

更新减少熵

GPS或视觉观测更新后，不确定性减小，信息增益为正。

⚠️

异常观测

如果观测的噪声模型不准确（如GPS信号被遮挡），更新反而会引入错误信息。

⚠️
Mahalanobis距离：判断一个新观测是否"可信"的标准方法。计算观测残差与预测协方差的Mahalanobis距离，如果超过阈值（如χ²分布的95%分位点），则判定为异常观测并降低其权重。这正是VIO和LIO-SAM处理外点的核心机制。

公式：d_M = √((z - ẑ)ᵀ · S⁻¹ · (z - ẑ))
其中 S = HPHᵀ + R 是"新息协方差"。如果 d_M > χ²(α, n) 则判定为外点。
🛠️ 动手实践

用Python实现一维贝叶斯递推

目标：用Python模拟贝叶斯递推，观察预测步如何增加方差（熵增），更新步如何减小方差（信息增益）。

import numpy as np
import matplotlib.pyplot as plt

# ============ 一维贝叶斯递推（传感器融合的"心脏"）===========
# 假设状态 x 是无人机的高度（米）
# 先验：高斯 N(mu_prior, sigma_prior²)
# 似然：高斯 N(z, sigma_sensor²)  — z 是传感器观测
# 后验 = 归一化(先验 × 似然) = 高斯 N(mu_posterior, sigma_posterior²)

def bayes_update(mu_prior, sigma_prior, z, sigma_sensor):
    """贝叶斯更新：先验 × 似然 → 后验（均为高斯）"""
    # 高斯 × 高斯 = 高斯（归一化后仍是高斯）
    # 后验精度 = 先验精度 + 似然精度
    precision_prior = 1.0 / sigma_prior**2
    precision_likelihood = 1.0 / sigma_sensor**2
    precision_posterior = precision_prior + precision_likelihood

    mu_posterior = (mu_prior * precision_prior + z * precision_likelihood) / precision_posterior
    sigma_posterior = np.sqrt(1.0 / precision_posterior)
    return mu_posterior, sigma_posterior

# ============ 模拟：5次GPS观测，高度真值 = 100.0m ============
np.random.seed(42)
true_height = 100.0
sensor_noise = 2.0  # GPS传感器标准差 2m

# 先验（初始信念）：只知道大概在 90-110m 之间
mu, sigma = 90.0, 10.0  # N(90, 10²)

observations = [101.2, 99.8, 100.5, 98.9, 100.1]
mu_history = [mu]
sigma_history = [sigma]

print("=== 贝叶斯递推：高度估计 ===")
print(f"初始: μ={mu:.2f}m, σ={sigma:.2f}m (3σ范围: {mu-3*sigma:.1f}~{mu+3*sigma:.1f}m)")
print()

for i, z in enumerate(observations):
    mu, sigma = bayes_update(mu, sigma, z, sensor_noise)
    mu_history.append(mu)
    sigma_history.append(sigma)
    print(f"观测{i+1}: z={z:.1f}m | 更新后: μ={mu:.2f}m, σ={sigma:.2f}m (3σ范围: {mu-3*sigma:.1f}~{mu+3*sigma:.1f}m)")

print(f"\n真实高度: {true_height}m")
print(f"最终估计: {mu:.2f}m (误差: {abs(mu-true_height):.2f}m)")

# ============ 可视化：不确定性如何随观测减少 ============
fig, axes = plt.subplots(1, 2, figsize=(13, 4.5))

# 左图：估计值收敛
ax = axes[0]
ax.axhline(true_height, color='green', lw=2, label='真实高度')
ax.plot(range(len(mu_history)), mu_history, 'bo-', lw=2, markersize=8, label='贝叶斯估计 μ')
ax.fill_between(range(len(mu_history)),
                np.array(mu_history) - 3*np.array(sigma_history),
                np.array(mu_history) + 3*np.array(sigma_history),
                alpha=0.2, color='blue', label='±3σ不确定区间')
ax.set_xlabel('观测次数'); ax.set_ylabel('高度 (m)')
ax.set_title('贝叶斯递推：估计值随观测收敛到真值')
ax.legend(); ax.grid(True, alpha=0.3)

# 右图：标准差衰减
ax = axes[1]
ax.plot(range(len(sigma_history)), sigma_history, 'rs-', lw=2, markersize=8)
ax.axhline(sensor_noise, color='orange', lw=2, linestyle='--', label=f'传感器噪声 σ={sensor_noise}m')
ax.set_xlabel('观测次数'); ax.set_ylabel('估计标准差 σ (m)')
ax.set_title('不确定性（σ）如何随观测次数递减')
ax.legend(); ax.grid(True, alpha=0.3)

plt.tight_layout()
# plt.savefig('bayes_filter_convergence.png', dpi=150)
plt.show()
关键观察：① σ 随观测次数呈 1/√n 速度衰减；② 第1次观测后σ从10m迅速降至约1.8m（大幅信息增益）；③ 第5次观测后σ约0.9m，接近传感器噪声水平——这正是GPS+IMU融合的理论基础。

⚠️
常见陷阱① - 错误假设高斯噪声：很多教科书假设传感器噪声是高斯分布，但实际上某些噪声是长尾分布（heavy-tailed）——如GPS多径误差、视觉特征匹配的误匹配。如果用高斯模型描述这些噪声，卡尔曼滤波会对异常观测过度响应，导致发散。解决：使用混合高斯模型（GM-PHD）或M估计器。

常见陷阱② - 混淆先验精度与似然精度：贝叶斯更新中，精度 = 1/σ² 是加法关系。如果先验σ=10m（精度0.01），似然σ=1m（精度1.0），则后验σ≈0.99m——似然完全主导了结果。如果先验σ=0.1m（精度100），则后验σ≈0.0995m——先验几乎不变。

常见陷阱③ - Mahalanobis距离阈值设置不当：χ²阈值的选择取决于自由度（状态维度）和置信水平。VIO中常用95%分位点——但这意味着每20个正常观测就有1个会被误判为外点！如果频繁触发，需要用更高阈值（99.7%）或自适应阈值。

概率滤波的统一框架

① 高斯分布：传感器噪声的合理近似模型，由均值和方差完全描述。
② 协方差矩阵：多维不确定性的表示，协方差传播 Σ_y = A·Σ_x·Aᵀ。
③ 贝叶斯滤波：预测步（增加不确定性）+ 更新步（减少不确定性）。
④ Mahalanobis距离：判断观测可信度的核心方法，超过χ²阈值判定为异常观测。
⑤ 信息熵：不确定性的量化度量，预测使熵增，更新使熵减，σ随观测次数呈1/√n衰减。
