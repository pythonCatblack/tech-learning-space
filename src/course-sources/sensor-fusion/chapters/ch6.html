前置依赖：Chapter 5（IMU漂移本质）建立了"为什么需要绝对位置"的动机。本章将回答GPS的核心问题——如何在没有累积误差的前提下实现全球绝对定位。IMU+GPS融合是第9章 Kalman滤波最经典、最入门的应用场景，第10章的因子图也通过GPS因子节点（绝对位置约束）来锚定全局坐标系。

GPS（Global Positioning System）是无人机能在户外获得"绝对、不漂移的全球坐标"的最核心手段。它的误差不随时间累积（这点与IMU完全相反），但接收条件苛刻——室内不可用、峡谷精度下降、强多云电离层干扰。理解了GPS，就理解了户外无人机定位的上限——以及为什么工业级作业还需要 VIO 和 LIO-SAM 来兜底。

6.1 GNSS基础：伪距方程与三角测量

6.1.1 四大全球导航系统

统称GNSS（Global Navigation Satellite System）——不止美国的GPS：

🇺🇸

GPS (美国)

31颗卫星，MEO轨道（~20,200km）。最早部署（1995年满编），全球覆盖最成熟。

🇨🇳

BDS/北斗 (中国)

30颗卫星，混合轨道（GEO+IGSO+MEO）。亚太区域精度最高，全球覆盖已完成。

🇪🇺

Galileo (欧盟)

24颗卫星，MEO轨道。民用定位精度与GPS相当，搜救转发功能独特。

🇷🇺

GLONASS (俄罗斯)

24颗卫星，MEO轨道。高纬度（靠近极地）精度优于GPS。

6.1.2 伪距方程：推算位置的数学模型

每颗卫星向接收机广播两样东西：①精确轨道参数（星历）和②信号发射时间戳。接收机对比到达时间与发射时间，算出伪距（Pseudorange）：

🔑 伪距方程（单颗卫星）

ρ = c · (t_receive - t_transmit) = |SV_pos - RX_pos| + c · Δt_rx + ε

其中：
• ρ：伪距（m），含所有误差
• c：光速 = 299,792,458 m/s
• |SV_pos - RX_pos|：卫星与接收机的真实几何距离
• c · Δt_rx：接收机时钟偏差（纳秒级即米级误差）
• ε：电离层延迟、对流层延迟、多径误差等的总和

为什么需要4颗以上卫星？未知量：3个坐标 (x, y, z) + 1个时钟偏差 (Δt) = 4个未知数。至少4颗卫星的伪距联立求解。

🔑 伪距方程组的矩阵写法

设第 i 颗卫星坐标为 (Sxᵢ, Syᵢ, Szᵢ)，已知它的伪距为 ρᵢ。对于接收机坐标 (x, y, z) 和时钟偏差项 cdT = c·Δt（未知，是所有卫星共用的）：

√((Sxᵢ-x)² + (Syᵢ-y)² + (Szᵢ-z)²) + cdT = ρᵢ

这是一个非线性方程组。——通常用 Newton-Raphson 迭代求解（先线性化在当前估计点，迭代收敛）。解这个方程组的过程，就是"GPS单点定位"（SPP, Single Point Positioning）。

6.2 从 5m → 2cm：差分增强技术

6.2.1 差分定位（DGPS/DGNSS）的直觉

核心思路：GPS误差源（电离层延迟、卫星钟差）对位置接近的两个接收机——几乎相同。利用"已知位置的基准站"来实时测算这些误差的影响，再通过通信链路播发给无人机。

6.2.2 RTK：厘米级实时定位的工程实现

RTK（Real-Time Kinematic）是目前工业无人机（DJI Matrice 350 RTK、5230B等测绘机型）的标配。

🔑 RTK完整工作链路

① 基准站（Base Station）：固定在已知精确坐标的位置（精度通常通过长时间静态 PPP 获得，坐标可提前部署）。

② 基准站计算实时"修正量"：基准站同时接收 GPS 信号，计算自己的"GPS 定位值"与其"已知精确坐标"之间的偏移——即当前时刻这组卫星的实时误差。

③ 实时播发修正量：通过4G/NB/电台/无人机内置的数据链路将修正量（RTCM 3.x 格式）发送给飞行中的无人机。

④ 无人机（流动站）应用修正：无人机接收这组修正量，从自己的GPS测量中去掉。剩余误差主要来自流动站与基准站的距离：基线距离越大，修正越不完整。RTK的有效基线距离通常 < 10-20km。

修正后定位精度：水平 ±1-2cm，垂直 ±2-3cm（双频 L1+L2 接收机）。需固件支持双频 + 多系统（GPS+BDS+Galileo+GLONASS）联合定位。

6.2.3 RTK实战部署清单

1

获取基准站坐标

在作业区域提前架设 GNSS 接收机（如 Trimble R10 或千寻魔盒），连续观测 10+ 小时，PPP 解算获取精确坐标。

2

配置 RTK 链路

基准站通过 DTU/4G DTU 发送 RTCM 3.2 格式的实时修正数据。DJI Matrice 350 支持直接接入千寻/六分等 CORS 服务的 NTRIP 数据流。

3

验收 RTK 精度

起飞前在已知标记点验证——无人机悬停在该标记上方 2m，记录系统给出的位置与标记坐标的偏差。偏差应 < 5cm。

4

建立融合链路

将 GPS+RTK 的定位结果送入 Kalman 滤波（第9章）。滤波器同时接收 IMU 的高频率数据（1kHz），以RTK的低频（通常 5-20Hz）作为绝对位置观测更新。

6.3 GPS+IMU融合：EKF的"Hello World"

这是传感器融合中最基础、最成熟、也最值得亲手实现的一个组合。GPS提供绝对位置观测（低频，约5-20Hz），IMU提供高频率姿态估计（1kHz+）。

🔑 GPS+IMU EKF 融合的状态向量

状态向量 x = [位置 (ENU系, 3维), 速度 (ENU系, 3维), 姿态四元数 (4维), 加速度计偏差 (3维), 陀螺仪偏差 (3维)]ᵀ

共 16 维（3+3+4+3+3=16）

预测步：IMU 传播（每 1ms 执行一次）
更新步：GPS 观测更新（约 5-20Hz，即每 50-200ms 一次）

这个融合系统的核心调参项：
① Q 矩阵（过程噪声）——从 IMU 的 Allan 方差（第5章）读取
② R 矩阵（GPS 观测噪声）——从 GPS 接收机的规格书读取

GPS+IMU EKF 的定位精度与 GPS 单机精度相当，但输出的位姿是平滑、连续的——而且 GPS 短暂失效（几秒）时，IMU 可以继续提供姿态估计。

6.4 GPS的失效边界与毫米波雷达的兜底

6.4.1 GPS失效的典型场景

🏙️

城市峡谷

高楼遮挡 + 大量多径反射 → 可用卫星骤减（< 4颗） → GPS精度从2cm退化到10m+或完全失效。

🌲

密林/隧道

GPS L1信号（1575.42MHz）被树冠完全遮挡。消费级接收机完全不可用。

🏠

室内/仓库

GPS完全失效。需切换至 VIO（第11章）或 UWB定位。

6.4.2 毫米波雷达：全天候感知的互补传感器

当GPS失效时，避障和导航需要其他传感器兜底。毫米波雷达（77GHz）穿透雨雾性能远优于 Lidar，已成为无人机避障的一个重要选项。

77GHz：雨雾穿透能力强，小雨衰减< 1dB/km FMCW多普勒测速：每个目标自带速度测量，无需位置差分 量产成本：单模组 $20-100（车载雷达产业链带动）
🛠️ 动手实践

用 Python 解析 GPS NMEA 数据并绘制轨迹

目标：读取真实GPS NMEA 日志，解析GPGGA/GPRMC句子，提取经纬度和高程，绘制飞行轨迹。

# 读取 NMEA 日志文件
import re

def parse_gpgga(line: str):
    """解析 GPGGA（全球定位系统固定数据）"""
    parts = line.split(',')
    if len(parts) < 10: return None
    lat_dm = parts[2]  # 度分格式：DDMM.MMMM
    lat_dir = parts[3]  # N/S
    lon_dm = parts[4]  # DDDMM.MMMM
    lon_dir = parts[5]  # E/W
    alt_msl = parts[9]   # 海拔高度（m）
   
   lat = float(lat_dm[:2]) + float(lat_dm[2:])/60.0
   lon = float(lon_dm[:3]) + float(lon_dm[3:])/60.0
   if lat_dir == 'S': lat = -lat
   if lon_dir == 'W': lon = -lon
   return lat, lon, float(alt_msl)

coords = []
with open('flight_log.nmea', 'r') as f:
   for line in f:
       if line.startswith('$GPGGA'):
           r = parse_gpgga(line)
           if r: coords.append(r)

print(f"解析到 {len(coords)} 个GPS点")

⚠️
常见陷阱① - RTK级联延迟：RTCM数据从基准站→4G→机载接收机，通常在50-200ms量级。如果这个时间延迟没有被 Kalman 滤波正确建模，位置估计会出现频谱分裂（高频成分消失而低频严重滞后）。在 PX4 中，RTK 延迟在 EKF2_EV_DELAY 参数中指定。

常见陷阱② - 忽视 GNSS 板块选择性：默认的 u-blox 模块通常只配置 GPS 或 GPS+GLONASS。在亚太地区飞行时，务必手动打开 BDS（北斗）——可显著增加可用卫星数（尤其在卫星仰角低的峡谷）。

常见陷阱③ - 在 GPS 失效边界上盲目切换：GPS 信号从7颗骤降到3颗时，位置估计会瞬跳到数米的误差。VIO/LIO-SAM 的接管必须有一个过渡区间（通常 3-5秒），在这个区间内两个传感器同时参与融合，确保平滑切换。
🎯 选型技巧

GNSS接收机选型——三个维度快速定位

维度1：单频 vs 双频——如果你需要 RTK 精度，必须选双频 (L1+L2/L5) 接收机。单频 RTK 的基线长度限制很短（< 5km），因为电离层延迟无法通过频率差消除。

维度2：是否支持外部RTK流入？——DJI Matrice 350 RTK 支持通过内置DTU接入千寻/六分的NTRIP服务；开源飞控（PX4+Pixhawk 6X）需要配置一个4G DTU直连GNSS接收机的CORR接口。

维度3：是否支持多系统 (GPS+BDS+Galileo+GLONASS)？——消费级无人机可能只配置单/双系统；专业测绘机应配置4系统，确保在任何地区（尤其高纬度——GLONASS；亚太——BDS）都能充分接收卫星。
