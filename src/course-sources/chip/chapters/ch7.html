CPU、GPU、FPGA、ASIC：
四种计算范式的终极对比

从通用到专用，从灵活到高效——理解为什么无人机最终需要 ASIC，以及 Taalas 为什么选择这条路线。

📖 约45分钟💻 架构对比
🔗
回顾 Chapter 6：上一章我们深入理解了现代 CPU 的复杂架构——流水线、超标量、乱序执行、分支预测、缓存一致性。这些机制让 CPU 能高效处理"任何程序"，但也付出了巨大的硬件和功耗代价。本章将展示：当任务足够明确时，放弃这些通用机制、走向专用化，可以获得数量级的效率提升。

CPU 是"瑞士军刀"——什么都能做，但什么都不精。GPU 是"美术刀阵"——擅长并行切割。FPGA 是"可重构模具"——能变成任何形状。ASIC 是"专用模具"——只为一个产品优化到极致。理解这四种计算范式的本质差异，是理解无人机硬件加速路径的钥匙。

7.1 CPU：通用计算的"瑞士军刀"

CPU 的设计哲学是"优化单线程延迟"（Optimize Single-Thread Latency）。它假设程序是串行的、充满条件分支的、数据访问模式不可预测的——因此配备了大量通用机制来应对这些不确定性。

7.1.1 CPU 的"通用税"

CPU 为了"通用性"付出了哪些代价？

🎯

分支预测

15-25%的晶体管用于预测程序走向。猜错一次惩罚10-20周期。

🔄

乱序执行

ROB、保留站、寄存器重命名——30-40%的晶体管用于打破指令顺序。

💾

大缓存

L1/L2/L3 缓存占芯片面积40-60%。缓存未命中时性能骤降。

🌐

缓存一致性

多核之间的 MESI 协议消耗总线带宽和功耗。

⚠️
关键洞察：对于 AI 推理这样的"数据流任务"（输入→层层计算→输出，没有分支、没有乱序、数据访问完全可预测），CPU 的所有通用机制都是纯粹的开销。分支预测器永远猜对（因为没有分支），乱序执行没有东西可乱序，大缓存不需要（数据流是顺序访问）。这就是为什么 ASIC 可以把效率提升到 CPU 的100倍以上。

7.2 GPU：并行计算的"美术刀阵"

GPU 的设计哲学是"优化吞吐量"（Optimize Throughput）。它假设任务可以被分解成成千上万个独立的子任务——比如渲染屏幕上的每个像素、计算神经网络中的每个神经元。

7.2.1 GPU 的架构：SIMT

GPU 使用SIMT（Single Instruction, Multiple Threads，单指令多线程）架构：

1

流式多处理器 (SM)

GPU 由数十到上百个 SM 组成，每个 SM 包含几十个 CUDA 核心（ALU）。

2

Warp 调度

32个线程组成一个 Warp，共享同一条指令。Warp 内的所有线程同时执行相同的指令，但处理不同的数据。

3

零开销线程切换

当某个 Warp 等待内存时，SM 立即切换到另一个就绪的 Warp——用"线程级并行"掩盖内存延迟。

无人机场景

为什么 GPU 不是无人机的最佳选择？

NVIDIA Jetson 系列（如 Jetson Orin）是无人机视觉 AI 的常用平台，但它有致命缺陷：功耗太高（15-60W）。对于续航30分钟的消费级无人机，电池容量通常只有5000mAh@11.1V（约55Wh），Jetson Orin alone 就能在1小时内耗尽全部电量。这就是为什么大疆等厂商最终走向自研 ASIC。

7.3 FPGA：可重构的"万能积木"

FPGA（Field-Programmable Gate Array，现场可编程门阵列）是"硬件软件化"的极致——它由大量可编程逻辑单元和可编程互连组成，你可以用硬件描述语言（HDL）定义任何数字电路，然后"烧录"到 FPGA 中。

7.3.1 FPGA 的内部结构

🔲

可编程逻辑单元 (LUT)

每个 LUT 是一个小型查找表（如6输入1输出），可以实现任意6输入布尔函数。成千上万个 LUT 组合成任何逻辑。

🔗

可编程互连

LUT 之间通过可编程开关矩阵连接。你可以定义任何连线方式——就像在空中搭建任意电路。

💾

块 RAM (BRAM)

片上分布式 SRAM，用于存储数据和中间结果。每个 BRAM 通常是 18Kb 或 36Kb。

⚡

DSP 切片

专用的乘加单元（MAC），用于加速数字信号处理和 AI 推理中的矩阵乘法。

7.3.2 FPGA 的独特价值

FPGA 在芯片开发流程中扮演"原型验证平台"的角色：

✅ 可重构：同一硬件可以运行不同算法 ✅ 开发周期短：几周即可验证架构 ✅ 流片前验证：在真实硬件上测试设计 ❌ 成本高：单颗 FPGA 数千美元 ❌ 功耗高：可编程互连消耗大量电力 ❌ 频率低：通常只能跑到100-300MHz

7.4 ASIC：专用集成电路的极致效率

ASIC（Application-Specific Integrated Circuit，专用集成电路）是为单一特定任务设计的芯片。它放弃了所有通用性，把每一颗晶体管都用在刀刃上。

7.4.1 ASIC 的效率来源

为什么 ASIC 比 CPU/GPU 高效100倍？

去除一切通用开销

① 没有分支预测：数据流固定，路径预先确定。
② 没有乱序执行：计算顺序在编译时确定，硬件只需按顺序执行。
③ 没有大缓存：数据流从输入到输出，不需要随机访问内存。
④ 没有缓存一致性：通常是单核或确定性多核，无需复杂协议。
⑤ 定制化数据通路：每个运算单元的位置和连接都为特定算法优化。
⑥ 低精度运算：AI 推理只需要 INT8/INT4，不需要 CPU 的64位浮点单元。

本章关键概念

四种计算范式的核心要点

① CPU（瑞士军刀）：优化单线程延迟，配备分支预测、乱序执行、大缓存。适合通用任务，但”通用税”高。
② GPU（美术刀阵）：优化吞吐量，SIMT 架构用并行掩盖延迟。适合并行任务，但功耗高。
③ FPGA（万能积木）：可重构硬件，LUT+BRAM+DSP。适合原型验证，介于灵活性和效率之间。
④ ASIC（专用模具）：为特定任务定制，效率最高。适合量产产品，但灵活性最低。
