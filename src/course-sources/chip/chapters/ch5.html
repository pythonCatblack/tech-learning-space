5.1 算术逻辑单元 (ALU)：芯片的"计算器"

ALU（Arithmetic Logic Unit，算术逻辑单元）是 CPU 的核心执行单元——它负责执行所有的算术运算（加、减、乘、除）和逻辑运算（与、或、非、异或）。

5.1.1 ALU 的内部结构

一个典型的32位 ALU 包含以下组件：

1

算术单元

包含加法器、减法器（通过补码把减法变成加法）、乘法器（ Booth 算法或阵列乘法器）。32位加法通常使用超前进位加法器（CLA）来减少延迟。

2

逻辑单元

包含 AND、OR、XOR、NOT 等逻辑运算电路。这些运算可以并行执行，由选择信号决定输出哪一个。

3

移位单元

实现左移、右移、循环移位。移位在二进制中相当于乘/除以2的幂，是极高效的运算。

4

比较单元

比较两个数的大小，输出零标志（Zero Flag）、进位标志（Carry Flag）、溢出标志（Overflow Flag）等状态位。

5.1.2 超前进位加法器 (CLA)：打破串行瓶颈

简单的行波进位加法器（Ripple-Carry Adder）把32个全加器串起来——第1位的进位输出连接到第2位的进位输入，依此类推。这种结构的延迟是32个门延迟，太慢了。

超前进位加法器（Carry-Lookahead Adder, CLA）通过"预测"进位来打破串行依赖：它同时计算所有位的"生成信号"（G = A AND B）和"传播信号"（P = A XOR B），然后用树状结构并行计算所有进位。32位 CLA 的延迟从32个门降低到约6个门——这是 CPU 能运行在 GHz 频率的关键之一。

🔗
跨章关联：CLA 的"树状并行"思想在芯片设计中无处不在——Chapter 6 中的缓存 Tag 比较、Chapter 7 中 GPU 的并行计算单元、Chapter 13 中 AI 加速器的脉动阵列，本质上都是这种"把串行变成并行"的优化策略。

5.2 寄存器文件：CPU 的"工作台"

寄存器文件（Register File）是 CPU 内部的一组高速存储单元——比内存快100倍以上，但容量极小（通常只有32个或64个32位寄存器）。它是 ALU 的"工作台"：运算数据从寄存器来，运算结果回寄存器去。

5.2.1 寄存器文件的物理实现

寄存器文件本质上是一个小型 SRAM（静态随机存取存储器）阵列：

📍

译码器

把寄存器编号（如 R5）译码为字线（Word Line），选中对应的寄存器行。

📤

读出电路

通过位线（Bit Line）将选中寄存器的值读出，经过灵敏放大器（Sense Amplifier）放大后输出。

📥

写入电路

通过写驱动器将新值写入选中寄存器的存储单元（通常是6晶体管 SRAM 单元）。

5.2.2 为什么寄存器比内存快100倍？

寄存器文件和内存（DRAM）都是存储器，但速度差异巨大，原因有三：

距离近：寄存器在 CPU 核心内部，信号只需走几微米 电路简单：SRAM 单元只需6个晶体管，读出无需刷新 并行访问：寄存器文件支持同时读2个、写1个寄存器

相比之下，DRAM 单元只有1个晶体管+1个电容，但需要定期刷新（电容漏电），且位于 CPU 外部的独立芯片上，信号要穿过 PCB 走线和封装引脚——距离远了几个数量级。

5.3 有限状态机 (FSM)：芯片的"决策大脑"

有限状态机（Finite State Machine, FSM）是数字电路中实现"决策逻辑"的核心结构。它由三部分组成：当前状态（存储在寄存器中）、下一状态逻辑（组合逻辑）、输出逻辑（组合逻辑）。

5.3.1 FSM 的两种类型

类型输出依赖特点典型应用
Mealy 型当前状态 + 输入输出响应更快，但可能产生毛刺通信协议控制器
Moore 型仅当前状态输出稳定，时序分析更简单CPU 控制单元

5.3.2 CPU 控制单元：最大的 FSM

CPU 的控制单元（Control Unit）本质上就是一个巨大的 FSM。它根据当前指令（状态）和时钟节拍，生成控制信号：ALU 做什么运算、寄存器文件读哪个地址、数据往哪里送、内存读还是写……

现代 CPU 的控制单元极其复杂——一条指令可能需要几十个时钟周期来完成（取指、译码、执行、访存、写回），每个周期控制单元都要输出数十个控制信号。这就是为什么 CPU 设计是芯片设计中最具挑战性的领域之一。

5.4 总线：芯片内部的"高速公路"

芯片内部有数十个模块需要相互通信——CPU、GPU、内存控制器、USB 控制器、显示引擎……它们通过总线（Bus）连接。总线是芯片内部的"高速公路系统"。

5.4.1 总线的三个组成部分

🛣️

地址总线

指明"和谁通信"——发送目标模块的地址编号。宽度决定可寻址空间（32位→4GB，64位→16EB）。

📦

数据总线

传输实际数据。宽度决定单次传输的数据量（64位总线一次传8字节）。

🚦

控制总线

协调通信时序——读/写信号、就绪信号、中断请求等。

5.4.2 总线仲裁：谁先用"高速公路"？

当多个模块同时想使用总线时，需要总线仲裁（Bus Arbitration）来决定优先级。常见的仲裁策略：

固定优先级：CPU 永远优先，DMA 次之 轮询：每个模块轮流使用，公平但效率低 最小剩余时间优先：快完成的模块优先，减少等待

5.5 存储器层次：金字塔结构

芯片需要存储器来保存程序和数据，但"快"和"大"是矛盾的——最快的存储器（寄存器）最小，最大的存储器（硬盘/SSD）最慢。解决方案是存储器层次结构（Memory Hierarchy）——用金字塔结构平衡速度、容量和成本。

寄存器 (Register) — 最快、最小、最贵 — ~1KB, ~1ns
L1 缓存 — 32-64KB, ~2ns
L2/L3 缓存 — 256KB-64MB, ~5-20ns
主内存 (DRAM) — 8-128GB, ~100ns
🔗
跨章关联：这个存储器层次结构是理解 Chapter 6（芯片架构）和 Chapter 7（CPU vs GPU vs ASIC）的关键。CPU 的设计哲学是"让缓存命中率达到99%"；GPU 的设计哲学是"用海量并行掩盖内存延迟"；而 ASIC（如 Taalas HC1）的设计哲学是"把数据流固定死，彻底消除缓存未命中的可能"。三种哲学，三种硬件形态。
