第六章：AI编码助手实战——ChatGPT与Copilot的高效使用

6.1 AI编码助手的定位

AI编码助手不是替代程序员，而是程序员的"第二双手"。它擅长处理重复性编码任务、探索未知API、提供多角度思路。但它也需要正确的引导才能发挥最大价值。

理解AI的能力边界是关键：AI擅长——模式化代码生成、代码解释与翻译、调试思路建议、文档生成。AI不擅长——完全替代业务理解、处理最新框架API（知识截止限制）、处理超长上下文、保证100%正确的代码。

3. 可能的性能瓶颈在哪里？
4. 如果要优化，你建议从哪个方向入手？

代码：
```typescript
function processTaskQueue(tasks: Task[]): ProcessedResult {
  const sorted = tasks.sort((a, b) => b.priority - a.priority);
  const groups = groupBy(sorted, 'status');

  return Object.entries(groups).reduce((acc, [status, items]) => {
    acc[status] = items.map(item => ({
      ...item,
      processedAt: new Date(),
      result: executeTask(item)
    }));
    return acc;
  }, {} as ProcessedResult);
}
```"

提示词：

1. 添加完整的类型定义
2. 使用类型守卫处理边界情况
3. 保持原有逻辑不变

代码：
```javascript
function calculatePriority(task) {
  const now = Date.now();
  const due = new Date(task.dueDate).getTime();
  const daysUntilDue = (due - now) / (1000 * 60 * 60 * 24);

  if (task.isBlocked) return 0;
  if (daysUntilDue < 0) return 100;
  if (daysUntilDue < 1) return 80;
  if (daysUntilDue < 3) return 60;

  return 40 + (task.importance * 10);
}
```"

Cursor是一个深度集成AI的IDE，采用AI-First设计理念，相比VS Code + Copilot有独特优势：

6.5 Prompt Engineering for Code

好的Prompt是获得高质量AI代码的关键。

GREENS原则：

  • Goal（目标）：明确告诉AI要做什么
  • Role（角色）：设定AI扮演的角色
  • Engineering context（工程上下文）：提供足够的代码上下文
  • NConstraints（约束条件）：说明限制和边界
  • Style（风格）：指定代码风格要求
提示词示例：
"作为TypeScript专家（Role），请帮我实现一个任务过滤函数（Goal）。

上下文：
- 我们使用TypeScript + React
- 已有Task类型定义
- 需要支持多条件组合过滤

约束：
- 不要使用any类型
- 保持函数纯粹（无副作用）
- 导出为默认导出

风格：
- 使用函数式编程风格
- 添加JSDoc注释
- 遵循airbnb编码规范

请提供完整实现和单元测试。"

6.6 常见陷阱与规避

  1. 过度依赖：不假思索接受AI建议，应先理解再使用
  2. 上下文丢失：长对话中AI可能遗忘早期上下文，注意清理和补充
  3. 知识截止：AI知识有截止日期，新框架/库可能不了解
  4. 安全风险：AI可能生成包含安全漏洞的代码（如SQL注入），必须审查

6.7 本章小结

  1. AI编码助手是程序员的效率工具，不是替代品
  2. ChatGPT适合代码解释、翻译、Bug诊断等多场景
  3. GitHub Copilot擅长上下文感知的实时补全
  4. GREENS原则能帮助写出高质量的AI生成Prompt
  5. 保持批判性思维，审查AI生成的每一行代码
