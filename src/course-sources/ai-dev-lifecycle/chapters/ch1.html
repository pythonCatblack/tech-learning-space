在传统软件开发中，一个产品从想法到上线通常经历：需求调研 → 架构设计 → 编码实现 → 测试验证 → 部署运维。每个环节都依赖人工决策，效率瓶颈明显。而 AI AGENT（智能体）的出现，正在从根本上改变这一范式。

1.1 课程全景地图

AI AGENT 不仅仅是"更聪明的代码补全工具"，它是一个能够理解目标、自主规划、调用工具、执行任务的智能系统。将其融入开发全周期，就形成了AI-Augmented Development（AI增强开发）新范式。

💡 AI AGENT 的核心能力

AI AGENT 的核心能力可以用"感知-规划-执行-反馈"四环来概括：感知环境（理解需求、读取代码）、规划步骤（分解任务、选择策略）、执行动作（生成代码、调用API）、反馈优化（评估结果、迭代改进）。

1.2 为什么是"全周期"？

很多团队引入AI只停留在"写代码"环节，这其实是最大的浪费。从产品创意阶段的头脑风暴，到需求分析时的用户故事提取，再到架构设计、代码生成、测试编写、部署发布、运维监控——AI AGENT在每个环节都能发挥独特价值。

举个具体例子：某团队开发一个电商推荐系统。传统流程需要产品经理写PRD、架构师画蓝图、开发者写代码、测试人员编写用例、运维人员配置集群。有了AI AGENT，这个流程可以变成：
